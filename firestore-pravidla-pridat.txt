PRAVIDLA FIRESTORE — CO PŘIDAT DO PROJEKTU nakupni-pozadavky
============================================================

Modul Dovolenky nově ukládá data do projektu nakupni-pozadavky, do kolekcí
"people" a "absences". Do STÁVAJÍCÍCH pravidel Firestore je proto potřeba přidat
tyto dva bloky (číst i zapisovat smí jen přihlášený uživatel):

Firebase konzole → projekt nakupni-pozadavky → Firestore Database → Rules.
Dovnitř bloku  match /databases/{database}/documents { ... }  vlož:

    // --- Dovolenky ---
    match /people/{id} {
      allow read, write: if request.auth != null;
    }
    match /absences/{id} {
      allow read, write: if request.auth != null;
    }

DŮLEŽITÉ:
- Nemaž stávající pravidla pro requests / users / meta — jen přidej tyto dva
  bloky vedle nich, dovnitř stejného match /databases/{database}/documents { }.
- Pokud tam máš na konci "catch-all" pravidlo, které vše ostatní zakazuje
  (match /{document=**} { allow read, write: if false; }), nech ho AŽ ÚPLNĚ
  NAKONEC, za těmito bloky.
- Po vložení klikni na Publish.
